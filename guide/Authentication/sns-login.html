# 외부 공급자 인증

Firebase 인증 방법 중 외부 공급자(예: Google, GitHub, Facebook)를 사용하는 방법을 학습합니다.

# Google 인증

  1. Firebase 로그인 방법 설정을 참고하여 Goggle 사용을 활성화 합니다.

  2. 컴포넌트 메서드 signInWithGoogle을 추가한 후, Google 인증을 팝업으로 띄우는 코드를 작성합니다. 인증이 성공할 경우, authUser 컴포넌트 데이터에 응답 받은 결과의 user를 할당합니다. 오류가 발생한 경우는 콘솔에 오류 메시지를 출력합니다.



 
 
 
 
 


export default {
  // ...
  signInWithGoogle() {
    const provider = new auth.GoogleAuthProvider();
    auth().signInWithPopup(provider)
      .catch(error => console.error(error.message));
  }
}

# GitHub 인증

  1. Firebase 로그인 방법 > GitHub 사용 설정을 누른 후, 콜백 URL을 복사합니다.
  1. GitHub 애플리케이션 등록 페이지로 이동한 다음 입력해야 할 정보를 기입합니다. 복사했던 콜백 URL은 Authentication callback URL 입력 필드에 붙여 넣습니다. Register application 버튼을 누르면 애플리케이션이 GitHub에 등록됩니다.

NOTE

테스트 목적이라면 홈 페이지 URL 값을 localhost로 사용하고, 실제 서비스 목적일 경우 서비스 페이지 URL을 입력합니다.

  1. 애플리케이션 등록이 완료되면 Client ID, Client Secret 코드가 발급되어 화면에 표시됩니다.
  1. 발급 받은 Client ID, Client Secret 코드를 복사하여 GitHub 사용 설정에 붙여 넣고 저장 버튼을 누릅니다.
  1. 애플리케이션에 GitHub 인증을 처리하는 메서드를 추가하고 코드를 작성합니다.


 
 
 
 
 
 

export default {
  // ...
  signInWithGitHub() {
    const provider = new auth.GithubAuthProvider();
    auth().signInWithPopup(provider)
      .catch(error => console.error(error.message));
  }
}

  1. GitHub 인증 메서드를 실행시키는 버튼을 만들어 클릭하면 GitHub 인증 팝업 창이 뜹니다. Authorize {사용자_계정} 버튼을 누르면 애플리케이션 로그인이 완료됩니다.

잠깐?

인증 허용된 애플리케이션은 GitHub 애플리케이션 설정에서 찾아 취소(Revoke)할 수 있습니다.

NOTE

보다 자세한 내용은 GitHub으로 인증하기 문서를 참고합니다.
