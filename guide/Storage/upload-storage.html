# 파일 업로드

이미지를 업로드 할 간단한 템플릿 코드를 작성합니다. file 인풋 컨트롤을 추가한 후 화면에서 감추고, 대신 이미지 업로드 버튼을 추가한 후 클릭하면 file 인풋 컨트롤을 클릭한 것처럼 시뮬레이션 합니다.

<div class="upload-demo">
  <div class="profile">
    <img
      ref="profile_image"
      :src="authUser.photoURL"
      :alt="authUser.displayName"
      style="height: 150px">
  </div>
  <div class="upload-progress-bar">
    <progress ref="progress" min="0" value="0" max="100"></progress>
  </div>
  <input ref="upload" type="file" @change="uploadFile" style="display: none">
  <button type="button" @click="$refs.upload.click()">이미지 업로드</button>
</div>

# 파일 이름

file 인풋 컨트롤 change 이벤트에 연결된 메서드 uploadFile을 정의한 후, 이벤트 객체로부터 업로드할 파일 객체를 가져와 파일 이름에 고유한 날짜 정보를 접합해 업로드할 파일 이름을 만듭니다.

methods: {
  uploadFile(e) {
    const file = e.target.files[0];
    const fileName = `${Date.now()}-${file.name}`;
    console.log(fileName); // 1548138370513-br-pint.jpg
  },
}

# put() 메서드

스토리지 참조의 put() 메소드를 사용하면 파일을 스토리지 참조에 업로드할 수 있습니다.




 


uploadFile(e) {
  const file = e.target.files[0];
  const fileName = `${Date.now()}-${file.name}`;
  const uploadTask = imagesRef.child(fileName).put(file);
}

# 메타데이터 추가

파일을 업로드할 때 파일의 메타데이터를 추가할 수도 있습니다. name, size, contentType(MIME 형식) 등 일반적인 파일 메타데이터 속성을 포함할 수 있습니다.




 
 
 


uploadFile(e) {
  const file = e.target.files[0];
  const fileName = `${Date.now()}-${file.name}`;
  const { name, size, type } = file;
  const metadata = { name, size, contentType: type };
  const uploadTask = imagesRef.child(fileName).put(file, metadata);
}

# 업로드 이후 처리

put() 메서드는 Promise를 반환하므로 then()을 사용해 프로세스가 처리된 이후 할 일을 설정할 수 있습니다. 업로드 이후 다운로드 가능한 URL 값을 사용하고자 한다면 다음과 같이 코드를 작성할 수 있습니다.







 
 
 


uploadFile(e) {
  const file = e.target.files[0];
  const fileName = `${Date.now()}-${file.name}`;
  const { name, size, type } = file;
  const metadata = { name, size, contentType: type };
  const uploadTask = imagesRef.child(fileName).put(file, metadata);
  uploadTask.then(snapshot => {
    snapshot.ref.getDownloadURL().then(url => console.log(url));
  });
}

# 업로드 진행률 모니터링

파일 업로드 중에 업로드 진행 상황을 감지하는 이벤트 state_changed를 사용해 진행중(progress), 실패(fail), 성공(success) 상태에 따라 콜백함수를 실행할 수 있습니다.





 
 
 
 
 
 
 
 
 
 
 


uploadFile(e) {
  // ...
  const uploadTask = imagesRef.child(fileName).put(file, metadata);

  uploadTask.on('state_changed',
    function progress(snapshot) {
      // 업로드 진행 중
    },
    function fail(error){
      // 업로드 실패
    },
    function success(){
      // 업로드 성공 및 완료
    }
  );
}

# 진행률 처리

파일 업로드 중인 진행 상황을 처리하는 콜백 함수 내에서 진행률(progress)을 구하는 코드를 작성합니다. 진행률은 진행 중인 시점의 바이트 수를 총 바이트 수로 나눈 후 100을 곱한 값입니다.

속성 설명
bytesTransferred 스냅샷을 생성한 시점까지 전송된 바이트(bytes) 수입니다.
totalBytes 업로드가 예정된 총 바이트(bytes) 수입니다.

진행률 % 값을 UI의 진행 표시바(<progress>) 요소의 값(value)로 설정하면 화면이 업데이트 됩니다. 업로드가 성공적으로 완료된 후, 프로필 이미지를 변경하려면 앞서 살펴본 getDownloadURL()를 통해 downloadURL을 반환 받는 코드를 추가합니다.






 

 





 
 
 




uploadFile(e) {
  // ...
  uploadTask.on('state_changed',
    snapshot => {
      // 진행률 % 구하기
      let progress = (snapshot.bytesTransferred / snapshot.totalBytes) * 100;
      // UI 프로그레스바 업데이트
      this.$refs.progress.value = progress;
    },
    error => {
      console.error(error.message);
    },
    () => {
      uploadTask.snapshot.ref.getDownloadURL().then(downloadURL => {
        this.$refs.profile_image.setAttribute('src', downloadURL);
      });
    }
  );
}
