# 초기화/배포

# 초기화

init 명령어를 실행하면 프로젝트 디렉토리의 루트에 firebase.json 구성 파일이 생성됩니다. 사이트를 초기화할 때 공개용 루트로 사용할 디렉토리를 지정하라는 메시지가 표시되며 기본값은 public 입니다. 공개용 루트 디렉토리에 올바른 index.html 파일이 없는 경우 이 파일이 자동으로 생성됩니다.


 

$ firebase login
$ firebase init  # 호스팅만 사용할 경우 명령어: firebase init hosting
1
2

# 프로젝트 추가

초기화 과정에서 프로젝트를 선택하지 않고 기본 프로젝트를 설정한 경우, 호스팅을 사용할 프로젝트를 추가할 수 있습니다.

$ firebase use --add
1

# 프로젝트 설정

프로젝트 별칭을 등록한 경우 use 명령을 사용해 별칭을 선택할 수 있습니다.

$ firebase use <프로젝트_별칭>
1

NOTE

별칭이 등록되었는지 확인하려면 아래 명령을 입력해보세요.

$ cat .firebaserc
1

# 배포

사이트를 배포하려면 프로젝트의 루트 디렉토리에서 deploy 명령어를 실행합니다. 콘텐츠가 Firebase 프로젝트의 기본 호스팅 사이트인 <your-firebase-project-id>.firebaseapp.com에 배포됩니다.

$ firebase deploy
1

# 오픈

Firebase 호스팅을 포함한 서비스를 명령어를 사용해 브라우저에 화면을 띄울 수 있습니다.

$ firebase open
1
