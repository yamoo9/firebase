# 데이터베이스 참조(ref)

데이터베이스에서 특정 참조 데이터를 가져오고자 한다면 ref() 메서드를 사용합니다. 예를 들어 chapters 데이터를 참조할 경우 아래와 같이 작성합니다.

import { database } from 'firebase';
const chaptersRef = database().ref('chapters');

# 하위 데이터 참조

Firebase 데이터베이스는 JSON 트리로 구성됩니다. 특정 데이터의 하위 데이터 아이템에 접근할 경우 다음의 방법을 사용할 수 있습니다.

database().ref('chapters/A109b4d');
1

또는 child() 메서드를 사용할 수도 있습니다.

database().ref('chapters').child('A109b4d');
1

특정 키(key) 값이 변수일 경우 다음과 같이 작성할 수 있습니다.

database().ref(`chapters/${chapterId}`);

// 또는

database().ref('chapters').child(chapterId);

# 읽기 이벤트

데이터베이스 참조 객체는 데이터의 변경 상태를 감지하는 이벤트를 설정할 수 있습니다.

  • value 이벤트 콜백
  • child_added 이벤트 콜백

# value 이벤트

value 이벤트는 읽기 이벤트 발생 시점에 특정 데이터베이스 참조 내용의 정적 스냅샷(snapshot)을 읽는데 사용됩니다. 이 이벤트는 초기 데이터가 확인될 때 1번 발생한 후 데이터가 변경될 때마다 발생합니다. 하위 데이터를 포함하여 해당 위치의 모든 데이터를 포함하는 스냅샷이 이벤트 콜백에 전달됩니다.

created() {
  chaptersRef.on('value', (snapshot) => {
    console.log(snapshot.val());
    // 콘솔 출력 (참조 객체 반환)
    //  {
    //    A10jbdt: {__ob__: Observer}
    //    A10koi1: {__ob__: Observer}
    //    A109b4d: {__ob__: Observer}
    //  }
  });
}

# child_added 이벤트

child_added 이벤트는 일반적으로 데이터베이스에서 데이터 아이템을 검색하는데 사용됩니다. 참조 데이터의 전체 내용을 반환하는 value와 달리 child_added는 기존 데이터 아이템마다 이벤트가 1번씩 발생한 후 지정된 참조에 데이터 아이템이 새로 추가될 때마다 이벤트가 다시 발생합니다. 새 데이터 아이템의 데이터를 포함한 스냅샷이 이벤트 콜백에 전달됩니다.

created() {
  chaptersRef.on('child_added', (snapshot) => {
    console.log(snapshot.val());
    // 콘솔 출력 (개별 데이터 아이템 반환)
    // { description: "Firebase는 인증, 실시간 데이터베이스 ...", title: "Firebase 소개" }
    // { description: "Firebase 인증(Authentication) ...", title: "Firebase 인증 소개" }
    // { description: "Firebase 실시간 데이터베이스는 ...", title: "Firebase 데이터베이스 소개" }
  });
}

# 컴포넌트 데이터 업데이트

value 이벤트를 사용할 경우, 참조 객체를 반환하므로 객체를 순환하는 for~in문을 사용하여 컴포넌트 데이터 chapters 배열 객체에 푸시(push)해야 합니다.







 
 
 
 
 
 
 




export default {
  data: () => ({
    chapters: []
  }),
  created() {
    chaptersRef.on('value', snapshot => {
      const chapters = snapshot.val();
      for ( let key in chapters ) {
        if ( chapters.hasOwnProperty(key) ) {
          const chapter = chapters[key];
          this.chapters.push(chapter);
        }
      }
    });
  }
}

반면 child_added 이벤트를 사용할 경우, 참조 객체의 개별 데이터 아이템을 반환하므로 별도로 객체를 순환할 필요 없이 컴포넌트 데이터 chapters 배열 객체에 푸시(push)할 수 있습니다.







 




export default {
  data: () => ({
    chapters: []
  }),
  created() {
    chaptersRef.on('child_added', snapshot => {
      this.chapters.push(snapshot.val());
    });
  }
}
