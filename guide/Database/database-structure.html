# 데이터 구조

모든 Firebase 실시간 데이터베이스 데이터는 JSON 객체로 저장됩니다. 데이터베이스를 클라우드 호스팅 JSON 트리라고 생각하면 됩니다. SQL 데이터베이스와 달리 테이블이나 레코드가 없으며, JSON 트리에 추가된 데이터는 연결된 키를 갖는 기존 JSON 구조의 노드가 됩니다.

Firebase 데이터베이스의 데이터는 의미를 가진 이름(예: author)이 키(key)가 될 수도 있고,

고유한 ID 값(예: A109b4d)이 키(Key)가 될 수도 있습니다.

# 내보내기

필요한 경우 Firebase 데이터베이스의 데이터를 JSON으로 내보낼 수 있습니다. (우측 상단의 more 버튼 클릭 시 화면에 표시)

내보낸 Firebase 데이터베이스 JSON 구조는 다음과 같습니다.

{
  "author" : "yamoo9",
  "chapters" : {
    "A109b4d" : {
      "description" : "Firebase는 인증, 실시간 데이터베이스, 스토리지, 호스팅, 클라우드 함수 등 다양한 기능을 제공하는 서비스입니다.",
      "title" : "Firebase 소개"
    }
  }
}

# 가져오기

JSON 파일을 가져와 Firebase 데이터베이스로 덮어쓰기 할 수 있습니다. (우측 상단의 more 버튼 클릭 시 화면에 표시)

JSON 가져오기 팝업 창이 화면에 표시되면 찾아보기를 눌러 JSON 파일을 찾은 다음 가져오기 버튼을 눌러 업로드할 수 있습니다.

chapters.json 파일을 다운로드 받은 후, 가져오기를 통해 Firebase의 데이터베이스를 덮어쓴다면 결과는 아래 이미지와 같아집니다.
