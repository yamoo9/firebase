# 데이터 지우기

Firebase 데이터베이스의 데이터를 삭제하는 가장 간단한 방법은 해당 데이터 참조의 remove() 메서드를 사용하는 것입니다.

chaptersRef.child('key').remove();
1

NOTE

set() 또는 update() 등 다른 메서드를 사용해 null을 지정하는 방법을 사용할 수도 있습니다.

# 데이터 참조 키

Firebase 데이터베이스의 특정 데이터 아이템을 제거하려면 데이터베이스에 저장된 키(key)가 필요합니다. 컴포넌트 생성 시점에 작성한 child_added 읽기 이벤트 코드를 다음과 같이 수정해 컴포넌트 chapters 데이터에 할당된 아이템마다 키 값을 id로 접근할 수 있도록 코드를 작성합니다.




 





created() {
  chatpersRef.on('child_added', snapshot => {
    this.chapters.push({
      id: snapshot.key,
      ...snapshot.val()
    });
  });
}

# 제거 메서드

사용자가 챕터 제거 버튼을 누르면 chapter 객체를 메서드에 전달해 제거하는 코드를 다음과 같이 작성할 수 있습니다.


 





 
 
 




<template>
  <button @click="removeChapter(chapter)">챕터 제거</button>
</template>

<script>
export default {
  methods: {
    removeChapter(chapter) {
      chaptersRef.child(chapter.id).remove();
    }
  }
}
</script>

챕터 제거 버튼을 누르면 Firebase 데이터베이스에서는 데이터가 제거되지만, 사용자 화면은 제거된 내용이 업데이트 되지 않습니다.

# 제거 이벤트 감지

Firebase SDK는 아이템이 제거될 때를 감지하는 이벤트 child_removed를 제공합니다. child_removed 이벤트를 사용하여 제거된 아이템을 찾아 인덱스를 확인한 후, 컴포넌트의 chapters에서 해당 아이템을 제거해 화면을 업데이트할 수 있습니다.





 
 
 
 
 


created() {
  chaptersRef.on('child_added', snapshot => {
    // ...
  });
  chaptersRef.on('child_removed', snapshot => {
    const removedChapter = this.chapters.find(chapter => chapter.id === snapshot.key);
    const removedIndex = this.chapters.indexOf(removedChapter);
    this.chapters.splice(removedIndex, 1);
  });
}
