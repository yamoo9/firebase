# 데이터 정렬/필터링

# 정렬

Firebase 데이터베이스는 키(key), 값(value) 또는 하위(child) 값으로 정렬된 데이터를 검색할 수 있습니다..

메서드 설명
orderByKey() 하위 키에 따라 결과를 정렬합니다.
orderByValue() 하위 값에 따라 결과를 정렬합니다.
orderByChild() 지정된 하위 키 또는 중첩된 하위 경로의 값에 따라 결과를 정렬합니다.

주의!

정렬 기준 메소드는 한번에 하나만 사용할 수 있습니다. 동일한 쿼리에서 정렬 기준 메소드를 여러 번 호출하면 오류가 발생합니다.

Error: Query.orderByChild: You can't combine multiple orderBy calls.

orderByChild() 메서드를 사용해 title, description 순으로 데이터를 정렬하는 코드는 다음과 같습니다.






 
 



import { database } from 'firebase';
const chaptersRef = database().ref('chapters');

export default {
  created() {
    let sortByTitle = chaptersRef.orderByChild('title'); // title 순으로 정렬
    let sortByDescription = chaptersRef.orderByChild('description'); // description 순으로 정렬
  }
}

정렬과 이벤트 구문을 같이 사용해 데이터를 컴포넌트의 데이터로 할당할 수 있습니다.


 









created() {
  chaptersRef.orderByChild('title').once('value', snapshot => {
    snapshot.forEach(data => {
      this.chapters.push({
        id: data.key,
        ...data.val()
      });
    });
  });
}

주의!

Firebase 데이터베이스는 내림차순 정렬 기능을 현재 제공하고 있지 않습니다. 다소 번거롭지만 Firebase 데이터베이스에서 오름차순으로 정렬된 데이터를 전송받으면 클라이언트 환경에서 내림차순 형태로 변경해야 합니다.


 





chaptersRef.orderByChild('metircs/likes').on('child_added', snapshot => {
  this.chapters.unshift({
    id: snapshot.key,
    ...snapshot.value()
  });
});

# 필터링

Firebase 데이터베이스는 개수에 제한을 두거나, 범위를 제공하는 메소드를 제공합니다.

메서드 설명
limitToFirst() 정렬된 결과 목록에서 맨 처음부터 반환할 최대 항목 개수를 설정합니다.
limitToLast() 정렬된 결과 목록에서 맨 끝부터 반환할 최대 항목 개수를 설정합니다.
startAt() 선택한 정렬 기준 메소드에 따라 지정된 키 또는 값보다 크거나 같은 항목을 반환합니다.
endAt() 선택한 정렬 기준 메소드에 따라 지정된 키 또는 값보다 작거나 같은 항목을 반환합니다.
equalTo() 선택한 정렬 기준 메소드에 따라 지정된 키 또는 값과 동일한 항목을 반환합니다.

NOTE

정렬 기준 메소드와 달리 여러 개의 제한 또는 범위 함수를 결합할 수 있습니다.

# 개수 제한

limitToFirst() 및 limitToLast() 메소드를 사용하여 특정 이벤트에서 동기화 할 하위 항목의 최대 개수를 설정할 수 있습니다. 예를 들어 limitToFirst()를 사용하여 제한을 10개로 설정할 경우 처음부터 10개까지 데이터를 제한하여 반환합니다. 데이터 개수가 10개가 안될 경우 모든 데이터를 반환합니다.

chaptersRef.limitToFirst(10).on('child_added', snapshot => {
  // ...
});

# 키 또는 값으로 필터링

startAt(), endAt(), equalTo() 메서드를 사용하여 쿼리의 시작, 종료 및 동일 지점을 임의로 선택해 데이터를 필터링 할 수 있습니다. 이 메소드는 데이터를 페이지 화하거나 하위 항목이 특정 값인 항목을 찾는데 유용합니다.

chaptersRef.startAt(2).endAt(7).on('child_removed', snapshot => {
  // ...
});

# 정렬 + 필터링 조합

Firebase 데이터베이스의 정렬, 필터링이 조합된 데이터를 처리할 수 있습니다.

chaptersRef
  .orderByChild('title') // 정렬
  .limitToLast(2)        // 필터링
  .once('child_added', snapshot => {
    // ...
  });
