# 데이터 쓰기

Firebase 데이터베이스에 새로운 데이터를 쓰는 방법은 set() 또는 push() 메서드를 사용합니다.

  • databaseRef.set() 메서드
    해당 참조의 기존 데이터를 모두 대체

  • databaseRef.push() 메서드
    새로운 데이터 아이템의 고유 키(key)를 생성하여 추가

# set 메서드

set() 메서드는 데이터베이스 참조의 모든 데이터를 덮어씁니다. (기존 데이터 삭제)

chaptersRef.set({
  "A1" : {
    "description" : "Firebase는 인증, 실시간 데이터베이스...",
    "title" : "Firebase 소개"
  },
  "A2" : {
    "description" : "Firebase 인증(Authentication) 서비스는...",
    "title" : "Firebase 인증 소개"
  },
  "A3" : {
    "description" : "Firebase 실시간 데이터베이스는...",
    "title" : "Firebase 데이터베이스 소개"
  }
});

chaptersRef.child('A1').set({
  description: 'Firebase용 Cloud Functions를 사용하면...',

# push 메서드

push() 메서드를 사용하면 데이터베이스 참조에 새롭게 추가할 데이터 아이템을 반환합니다.

const newChapter = chaptersRef.push();
1

반환된 아이템에 set() 메서드를 사용해 새로운 데이터 아이템을 설정할 수 있습니다.

newChapter.set({
  "description" : "Firebase의 Cloud Storage는...",
  "title" : "Firebase 스토리지 소개"
});
1

3
4

메서드 체이닝을 사용해 아래와 같이 구문을 연이어 작성할 수도 있습니다.

chaptersRef.push().set({
  "description" : "Firebase의 Cloud Storage는...",
  "title" : "Firebase 스토리지 소개"
});
1

프로그래밍 과정에서 push()와 set()을 연이어 호출하는 패턴이 자주 일어나므로 Firebase SDK는 다음과 같이 설정할 데이터를 push()에 직접 전달하여 두 메소드를 결합할 수 있습니다.

chaptersRef.push({
  "description" : "Firebase의 Cloud Storage는...",
  "title" : "Firebase 스토리지 소개"
});

컴포넌트 메서드 addChapter를 사용하여 Firebase 데이터베이스에 새로운 아이템을 쓰고자 한다면 다음과 같이 메서드 코드를 구성할 수 있습니다.











 




methods: {
  data: () => ({
    chapters: [],
    chapter: {
      title: '',
      description: ''
    }
  }),
  addChapter() {
    // chaptersRef.push().set({ ...this.chapter })과 동일
    chaptersRef.push({ ...this.chapter });
    this.chapters = { title: '', description: '' };
  }
}
