# 보안 규칙

데이터베이스 생성 시, 보안 규칙을 "잠금 모드로 시작" 했다면 데이터베이스는 비공개가 됩니다. 보안 규칙 설정을 변경하고자 한다면 Database > 규칙 탭을 선택한 후, 규칙의 .read, .write 값을 변경합니다.

# 비공개 설정

.read, .write 설정 값이 false일 경우 데이터베이스는 비공개 상태입니다.

# 모두 공개 설정

데이터베이스를 공개 상태로 변경하려면 .read, .write 설정 값을 true로 설정한 다음 게시 버튼을 누릅니다.

주의!

실 서비스에서 데이터베이스가 인증 없이 모두에게 공개되면 보안 문제가 발생됩니다.

NOTE

보다 자세한 내용은 Firebase 실시간 데이터베이스 규칙 이해 문서를 참고하세요.

# 특정 데이터 규칙 설정

데이터베이스의 특정 참조의 규칙을 설정하고자 한다면 아래와 같이 작성할 수 있습니다.

{
  "rules": {
    "chapters": {
      ".read": true,
      ".write": "auth !== null",
      ".indexOn": ["title", "description"]
    }
  }
}
